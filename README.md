# 基于Advance RAG的智能知识库系统

## 产品概述

《基于Advance RAG的智能知识库系统》是一款基于RAG（检索增强生成）技术的企业级智能知识库问答系统。该系统能够将企业文档转化为可智能检索的知识库，通过大语言模型为用户提供精准、高效的问答服务。

---

## 核心功能

### 1. 智能文档管理
- **多格式文档支持**：支持 Word (.docx)、PDF、CSV、Markdown (.md)、Excel (.xlsx) 等多种格式
- **自动分块处理**：采用智能文本分割技术，将长文档切分为适合检索的语义片段
- **向量化存储**：使用先进的Embedding技术将文档内容转化为向量，存储于ChromaDB向量数据库
- **多知识库管理**：支持创建和管理多个独立的知识库文档集合

### 2. 智能问答系统
- **语义检索**：基于向量相似度检索，精准定位相关问题答案
- **混合检索**：支持向量检索与关键词检索（BM25）混合使用
- **可调检索权重**：支持调整向量检索和关键词检索的权重比例
- **检索重排序**：支持对检索结果进行智能重排序，提升相关性
- **自动路由检索**：基于LLM自动判断用户问题应路由到哪个知识库
- **上下文增强**：自动整合检索到的相关文档片段，构建丰富的上下文提示
- **大模型生成**：调用通义千问、DeepSeek等多种大语言模型

### 3. Web交互界面
- **用户认证**：简单的工号登录系统
- **聊天历史**：支持保存和加载用户的问答历史
- **文档管理**：便捷的上传、切换和删除企业文档
- **检索参数配置**：实时调整检索数量、混合检索、重排序等参数

## 产品特性

### 1. 高精度语义检索
- 采用先进的Embedding技术，将文本转化为高维向量
- 支持Top-N相似度检索，精准定位相关内容
- 支持向量+关键词混合检索，适应不同检索场景

### 2. 灵活检索策略
- **混合检索**：融合向量相似度和关键词匹配
- **检索权重调整**：0-100%调整向量检索权重
- **结果重排序**：使用LLM进行结果重排序提升相关性
- **自动路由**：智能判断问题应路由到哪个知识库

### 3. 多知识库支持
- 支持上传和管理多个文档
- 每个文档独立的知识库集合
- 快速切换当前查询的知识库
- 文档删除时自动清理相关向量数据

### 4. 中文优化
- 针对中文文档特别优化
- 中文文件名自动处理为合规集合名
- 使用jieba等中文分词工具

### 5. 数据持久化
- 向量数据本地持久化存储于 `./chroma` 目录
- 聊天历史可选择本地文件或Redis存储
- 路由索引自动保存到 `route_index.json`

## 快速开始

### 环境要求
- Python 3.11+
- 阿里云API Key（用于访问通义千问模型）

### 安装步骤

1. **安装依赖包**
```bash
pip install -r requirements.txt
```

2. **配置API Key**
```bash
# 设置环境变量
export DASHSCOPE_API_KEY=your_api_key_here
```

3. **启动服务**
```bash
# 本地访问（仅限本机）
uvicorn main:app --host 127.0.0.1 --port 5000

# 局域网访问（手机等设备可访问）
uvicorn main:app --host 0.0.0.0 --port 5000
```

4. **访问系统**
打开浏览器访问：http://127.0.0.1:5000

### 使用流程

1. **登录**
   - 访问登录页面
   - 输入工号登录

2. **上传文档**
   - 访问文档上传页面
   - 选择支持的文档格式上传
   - 系统自动解析并建立向量索引

3. **选择知识库**
   - 在聊天界面下拉菜单选择知识库
   - 或启用自动路由让系统自动判断

4. **开始问答**
   - 输入问题，系统返回智能回答
   - 可调整检索参数（top_k、混合检索、重排序等）
   - 查看引用的文档来源

## API接口说明

### 1. 登录

**GET** `/login/`
- 显示登录页面

**POST** `/login/`
- 处理登录请求
- 请求体: `{"employee_id": "工号"}`
- 返回: 设置session_id cookie

---

### 2. 文档上传

**GET** `/document_upload/`
- 显示文档上传页面

**POST** `/document_upload/`
- 上传文档到知识库
- Content-Type: `multipart/form-data`
- 参数: `file` - 文档文件
- 支持格式: .docx, .pdf, .md, .xlsx, .csv, .txt, .doc
- 返回: JSON
```json
{
    "message": "文件上传成功",
    "filename": "文档名.docx",
    "collection_name": "文档名_hash"
}
```

---

### 3. 智能问答

**GET** `/` 或 **GET** `/chat/`
- 显示聊天页面

**POST** `/chat/`
- 处理问答请求
- Content-Type: `application/json`
- 请求体:
```json
{
    "message": "用户问题",
    "top_k": 10,
    "hybrid_search": false,
    "vector_weight": 0.5,
    "rerank": false,
    "rerank_top_k": 5,
    "auto_route": false,
    "session_id": "工号",
    "collection_name": "文档名_hash"
}
```
- 返回:
```json
{
    "response": "AI回答",
    "search_results": ["检索到的相关文档片段"]
}
```

---

### 4. 知识库管理

**GET** `/collection/`
- 获取所有文档列表和当前知识库
- 返回:
```json
{
    "name_list": ["文档1.docx", "文档2.pdf"],
    "collection_name": "当前知识库名"
}
```

**POST** `/collection/`
- 切换当前知识库
- 请求体: `{"collection_name": "文档名"}`

---

### 5. 文档管理

**GET** `/documents/`
- 获取所有已上传文档列表

**DELETE** `/document/`
- 删除文档及其向量数据
- 参数: `filename` - 要删除的文件名

---

### 6. 聊天历史

**GET** `/chat_history/?session_id=工号`
- 获取指定用户的聊天历史

---

### 7. 模型健康检查

**GET** `/api/model-health`
- 检查AI模型连接状态

## 配置说明

### 环境变量

| 变量名 | 说明 | 示例 |
|--------|------|------|
| `DASHSCOPE_API_KEY` | 阿里云API Key | `sk-xxxxxxxx` |
| `DEEPSEEK_API_KEY` | DeepSeek API Key（可选） | `sk-xxxxxxxx` |

### 检索参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `top_k` | int | 10 | 返回的最相关结果数量 |
| `hybrid_search` | bool | false | 是否启用混合检索 |
| `vector_weight` | float | 0.5 | 向量检索权重 (0-1) |
| `rerank` | bool | false | 是否启用重排序 |
| `rerank_top_k` | int | 5 | 重排序后保留的结果数 |
| `auto_route` | bool | false | 是否启用自动路由 |

## 注意事项

1. **向量数据库持久化**：数据存储在 `./chroma` 目录，删除该目录会丢失所有索引
2. **知识库命名**：文件名会自动清理为符合ChromaDB规范的名称（去除中文、特殊字符等）
3. **大文件处理**：大文档可能需要较长时间处理，请耐心等待
4. **并发安全**：已实现请求级别的并发安全处理
